📄 HTML5 — HyperText Markup Language

Foundation | Must Learn 100%
1

HTML Kya Hai? (Introduction)

Importance:
100% — Sabse Important

Logic Samjho:

HTML ek Markup Language hai jo web pages ka structure define karti hai. Iske bina koi bhi website nahi ban sakti. Ye browser ko batata hai ki kya heading hai, kya paragraph hai, kya image hai. HTML5 latest version hai jo 2026 mein sab jagah use hota hai.

Real Life Example:

Socho ek building ban rahi hai. Pehle structure (pillars, walls, beams) bante hain — wahi HTML hai. Uske baad paint aur decoration (CSS) hoti hai, aur finally electricity/water connection (JavaScript) hota hai. Bina structure ke decoration ka koi matlab nahi!

<!-- Basic HTML5 Document Structure -->
<!DOCTYPE html>
<html lang="hi">
  <head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Meri Pehli Website</title>
  </head>
  <body>
    <h1>Hello World!</h1>
    <p>Ye mera pehla paragraph hai.</p>
  </body>
</html>
Tip: Always DOCTYPE declare karo — ye browser ko batata hai ki ye HTML5 document hai. Bina iske browser quirks mode mein chalega jo unpredictable results dega.

CSS Kaise Kaam Karta Hai? (Basics)

Importance:
100% — Foundation

Logic Samjho:

CSS = HTML elements ko style karna. 3 tareeke hain CSS add karne ke:
1. Inline — Direct tag mein (style attribute) — Specific but messy
2. Internal — <style> tag mein head section mein — Single page ke liye
3. External — Alag .css file (Sabse Best — Reusable, Cacheable, Maintainable)

Real Life Example:

HTML = Ghar ka naksha (blueprint)
CSS = Paint, furniture, decoration
Inline = Ek room ka alag paint (specific but inconsistent)
External = Poori building ka same theme (consistent, easy to change)

/* ===== EXTERNAL CSS (style.css) — SABSE BEST ===== */

/* Element Selector — Saare paragraphs pe apply */
p {
  color: #334155;
  font-size: 1rem;
  line-height: 1.8;
}

/* Class Selector — Multiple elements pe apply */
.btn {
  background: #2563eb;
  color: white;
  padding: 12px 24px;
  border-radius: 8px;
  border: none;
  cursor: pointer;
}

/* ID Selector — Sirf ek unique element pe */
#main-header {
  background: linear-gradient(135deg, #667eea, #764ba2);
  padding: 60px 20px;
}

/* Descendant Selector */
nav a {
  color: white;
  text-decoration: none;
}

/* Child Selector */
ul > li {
  list-style: none;
}

/* Attribute Selector */
input[type="email"] {
  border: 2px solid #e2e8f0;
}

/* Pseudo-class */
.btn:hover {
  background: #1d4ed8;
  transform: translateY(-2px);
}

/* Pseudo-element */
p::first-line {
  font-weight: bold;
}

Box Model — Sabse Important Concept

Importance:
100% — Har Layout ka Base

Logic Samjho:

Har HTML element ek box hai. Box Model = Content + Padding + Border + Margin
Content = Andar ka maal (text, image)
Padding = Content aur border ke beech ka gap (andar ka space)
Border = Box ki deewar
Margin = Box ke bahar ka space (dusre boxes se doori)

Real Life Example:

Ek dabba socho jisme photo rakhi hai:
Photo = Content
Bubble wrap around photo = Padding (andar ka protection)
Cardboard box = Border (deewar)
Space between this box and other boxes = Margin (doori)

/* ===== BOX MODEL ===== */

.box {
  /* CONTENT */
  width: 300px;
  height: 200px;

  /* PADDING — Andar ka space (content se border tak) */
  padding: 20px;                    /* Sab taraf 20px */
  padding: 10px 20px;               /* Top-Bottom 10px, Left-Right 20px */
  padding: 10px 20px 15px 25px;     /* Top Right Bottom Left (clockwise) */

  /* BORDER — Box ki deewar */
  border: 3px solid #2563eb;        /* width style color */
  border-radius: 15px;              /* Corner round karna */
  border-top: 5px dashed red;       /* Sirf top border alag */

  /* MARGIN — Bahar ka space */
  margin: 30px auto;                /* auto = center horizontally */
  margin-top: 20px;
  margin: 10px 15px 20px 25px;      /* Top Right Bottom Left */

  /* ===== BOX-SIZING — SABSE IMPORTANT ===== */
  box-sizing: border-box;
  /* content-box (default): width = sirf content
    border-box: width = content + padding + border
    Hamesha border-box use karo!
  */
}

/* ===== DISPLAY PROPERTIES ===== */
.block { display: block; }           /* Poori width lega (div, p, h1) */
.inline { display: inline; }         /* Sirf content jitni width (span, a) */
.inline-block { display: inline-block; } /* Inline + width/height set kar sakte hain */
.none { display: none; }             /* Hide karna */
Warning: Hamesha box-sizing: border-box use karo — ye layout calculations easy kar deta hai! Bina iske width sirf content ki hoti hai, padding aur border alag add hote hain jo calculations mushkil banate hain.

Flexbox — Modern Layout King

Importance:
100% — 1D Layouts ke liye Best

Logic Samjho:

Flexbox = 1-dimensional layout system (row ya column). Parent ko display: flex karo, aur children automatically align honge. Ye responsive design ka base hai. Har modern website mein flexbox use hota hai.

Real Life Example:

Ek bookshelf socho jisme books automatically adjust hoti hain — chhoti jagah mein chhoti books, badi jagah mein badi books. Flexbox waise hi elements ko available space ke hisaab se adjust karta hai.

/* ===== FLEXBOX PARENT (Container) ===== */
.container {
  display: flex;

  /* ===== FLEX-DIRECTION ===== */
  flex-direction: row;              /* Default: left to right */
  /* flex-direction: row-reverse; */ /* Right to left */
  /* flex-direction: column; */     /* Top to bottom */
  /* flex-direction: column-reverse; */ /* Bottom to top */

  /* ===== FLEX-WRAP ===== */
  flex-wrap: wrap;                  /* Items next line pe jaayenge */
  /* flex-wrap: nowrap; */          /* Default: ek line mein force karega */
  /* flex-wrap: wrap-reverse; */   /* Reverse order mein wrap */

  /* Shorthand: flex-flow: direction wrap; */
  flex-flow: row wrap;

  /* ===== JUSTIFY-CONTENT — Main Axis (horizontal) ===== */
  justify-content: center;            /* Center mein */
  /* justify-content: flex-start; */   /* Start mein (default) */
  /* justify-content: flex-end; */     /* End mein */
  /* justify-content: space-between; */ /* Pehla start, aakhri end, beech mein equal */
  /* justify-content: space-around; */  /* Sab taraf equal space */
  /* justify-content: space-evenly; */  /* Har item ke beech exactly equal */

  /* ===== ALIGN-ITEMS — Cross Axis (vertical) ===== */
  align-items: center;                /* Vertically center */
  /* align-items: stretch; */         /* Default: full height */
  /* align-items: flex-start; */       /* Top pe */
  /* align-items: flex-end; */         /* Bottom pe */
  /* align-items: baseline; */        /* Text baseline pe align */

  /* ===== ALIGN-CONTENT — Multi-line alignment ===== */
  align-content: space-between;       /* Multiple lines ke beech space */
  /* align-content: center; */
  /* align-content: flex-start; */
  /* align-content: flex-end; */
  /* align-content: space-around; */
  /* align-content: stretch; */         /* Default */

  /* ===== GAP — Items ke beech gap ===== */
  gap: 20px;                         /* Row aur column dono mein */
  row-gap: 15px;                     /* Sirf rows mein */
  column-gap: 25px;                  /* Sirf columns mein */
}

/* ===== FLEXBOX CHILDREN (Items) ===== */
.item {
  /* ===== FLEX-GROW ===== */
  flex-grow: 1;                     /* Extra space ko kaise divide kare */
  /* flex-grow: 0; */                /* Default: grow nahi karega */

  /* ===== FLEX-SHRINK ===== */
  flex-shrink: 0;                   /* Chhoti screen pe shrink kare ya nahi */
  /* flex-shrink: 1; */              /* Default: shrink allowed */

  /* ===== FLEX-BASIS ===== */
  flex-basis: 200px;                /* Default size before grow/shrink */

  /* ===== SHORTHAND ===== */
  flex: 1 0 200px;                  /* grow shrink basis */
  /* flex: 1; */                    /* grow=1, shrink=1, basis=0% */

  /* ===== ALIGN-SELF ===== */
  align-self: flex-end;             /* Individual item alignment */
  /* align-self: center; */
  /* align-self: flex-start; */
  /* align-self: stretch; */

  /* ===== ORDER ===== */
  order: 2;                         /* Visual order change (default: 0) */
}
Tip: Flexbox mein gap property use karo — margin se behtar hai kyunki last item pe extra margin nahi aata. gap sirf items ke beech space देता hai.

CSS Grid — 2D Layout Powerhouse

Importance:
95% — Complex Layouts ke liye

Logic Samjho:

CSS Grid = 2-dimensional layout system (rows + columns dono). Complex layouts jaise dashboards, photo galleries, aur page layouts ke liye best hai. Flexbox 1D hai (sirf row ya column), Grid 2D hai (rows aur columns dono ek saath).

Real Life Example:

Excel spreadsheet socho — rows aur columns ka combination. Grid bhi waise hi kaam karta hai. Flexbox = ek line mein books arrange karna, Grid = chess board jaise arrange karna.

/* ===== CSS GRID PARENT ===== */
.grid-container {
  display: grid;

  /* ===== COLUMNS DEFINE KARNA ===== */
  grid-template-columns: 200px 200px 200px;   /* 3 fixed columns */
  grid-template-columns: 1fr 1fr 1fr;         /* 3 equal flexible columns */
  grid-template-columns: repeat(3, 1fr);      /* Same as above — shorthand */
  grid-template-columns: 200px 1fr 2fr;        /* Fixed + Flexible mix */

  /* ===== RESPONSIVE GRID — AUTO-FIT + MINMAX ===== */
  /* Ye sabse behtar tarika hai responsive grid banane ka! */
  grid-template-columns: repeat(auto-fit, minmax(250px, 1fr));
  /* auto-fit: Jitni jagah hai utni columns
    minmax(250px, 1fr): Minimum 250px, maximum equal share
    Screen chhoti hogi toh columns kam ho jayengi
  */

  /* ===== ROWS DEFINE KARNA ===== */

Logic Samjho:

Real Life Example:

Ek restaurant mein socho — normal tables = static position. VIP section alag jagah pe = absolute. Waiter table ke paas khada = fixed. Menu card table pe = relative. Stacking = plates ek ke upar ek.

  position: static;                 /* Default — normal flow mein */
  /* top, left, right, bottom — kaam nahi karenge */
}

  position: relative;               /* Normal position se shift */
  top: 20px;                       /* Neeche 20px (apni original position se) */
  left: 30px;                      /* Right 30px */
  /* Original space reserved rehta hai */
}

.absolute { 
  position: absolute;             /* Nearest positioned parent se relative */
  top: 0;                          /* Parent ke top se 0px */
  right: 0;                        /* Parent ke right se 0px */
  /* Parent ko position: relative; karna padta hai */
}

.fixed { 
  position: fixed;                /* Viewport (screen) se fixed */
  bottom: 20px;                    /* Screen ke bottom se 20px */
  right: 20px;                     /* Screen ke right se 20px */
  /* Scroll ke saath move nahi hoga */
}

.sticky { 
  position: sticky;               /* Scroll ke saath stick hota hai */
  top: 0;                         /* Top pe chipak jayega jab scroll karega */
  /* Parent ke andar sticky rehta hai */
}

/* ===== Z-INDEX — STACKING ORDER ===== */
.modal {
  position: fixed;
  top: 0; left: 0;
  width: 100%; height: 100%;
  background: rgba(0,0,0,0.5);
  z-index: 1000;                   /* Sabse upar dikhana hai */
}

.tooltip {
  position: absolute;
  z-index: 100;                    /* Modal se neeche, baaki sab se upar */
}

localStorage = Permanent (browser band karne ke baad bhi rahega, 5-10MB)
sessionStorage = Tab band hone tak (temporary, 5-10MB)
Cookies = Server ke saath bhejte hain, expiry date hoti hai (4KB)

Real Life Example:

localStorage = Ghar ka almari — permanent storage, kabhi saaf nahi hoti
sessionStorage = Hotel room ka drawer — check-out ke baad saaf ho jati hai
Cookies = ID card — har jagah dikhana padta hai, expiry date hoti hai

// Objects mapping

Error Handling & Debugging

Importance:
90% — Professional Code ke liye

Logic Samjho:

Errors hamesha aayengi — important hai unhe gracefully handle karna. try-catch se errors ko pakad ke user ko proper message dikhate hain bina page crash kiye. Debugging = Errors dhoondhna aur fix karna. Console methods aur DevTools isme madad karte hain.

Real Life Example:

try-catch = Car mein airbag. Accident hoga (error aayega) lekin airbag bachayega (catch block). Bina airbag ke accident fatal ho sakta hai (page crash).
Debugging = Doctor ki diagnosis — pehle symptoms dekho (console errors), phir test karo (breakpoints), phir treatment do (fix code).

/* ===== TRY-CATCH MODULES ===== */
try {
  let res = riskyOperation();
} catch (error) {
  console.error("Caught error:", error.message);
} finally {
  console.log("Execution structural cleanup done.");
}
Tip: Chrome DevTools mein Sources tab mein breakpoints lagao — code line-by-line execute karke debug karo. Network tab mein API calls check karo. Console mein $0 se currently selected element access kar sakte hain.
