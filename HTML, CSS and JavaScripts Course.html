Foundation | Must Learn 100%

HTML Kya Hai? (Introduction)

Importance:
100% — Sabse Important

Logic Samjho:

HTML ek Markup Language hai jo web pages ka structure define karti hai. Iske bina koi bhi website nahi ban sakti. Ye browser ko batata hai ki kya heading hai, kya paragraph hai, kya image hai. HTML5 latest version hai jo 2026 mein sab jagah use hota hai.

Real Life Example:

Socho ek building ban rahi hai. Pehle structure (pillars, walls, beams) bante hain — wahi HTML hai. Uske baad paint aur decoration (CSS) hoti hai, aur finally electricity/water connection (JavaScript) hota hai. Bina structure ke decoration ka koi matlab nahi!

<!-- Basic HTML5 Document Structure --> <!DOCTYPE html> <html lang="hi"> <head> <meta charset="UTF-8"> <meta name="viewport" content="width=device-width, initial-scale=1.0"> <title>Meri Pehli Website</title> </head> <body> <h1>Hello World!</h1> <p>Ye mera pehla paragraph hai.</p> </body> </html>
Tip: Always DOCTYPE declare karo — ye browser ko batata hai ki ye HTML5 document hai. Bina iske browser quirks mode mein chalega jo unpredictable results dega.

CSS Kaise Kaam Karta Hai? (Basics)

Importance:
100% — Foundation

Logic Samjho:

CSS = HTML elements ko style karna. 3 tareeke hain CSS add karne ke:
1. Inline — Direct tag mein (style attribute) — Specific but messy
2. Internal — <style> tag mein head section mein — Single page ke liye
3. External — Alag .css file (Sabse Best — Reusable, Cacheable, Maintainable)

Real Life Example:

HTML = Ghar ka naksha (blueprint)
CSS = Paint, furniture, decoration
Inline = Ek room ka alag paint (specific but inconsistent)
External = Poori building ka same theme (consistent, easy to change)

Box Model — Sabse Important Concept

Importance:
100% — Har Layout ka Base

Logic Samjho:

Har HTML element ek box hai. Box Model = Content + Padding + Border + Margin
Content = Andar ka maal (text, image)
Padding = Content aur border ke beech ka gap (andar ka space)
Border = Box ki deewar
Margin = Box ke bahar ka space (dusre boxes se doori)

Real Life Example:

Ek dabba socho jisme photo rakhi hai:
Photo = Content
Bubble wrap around photo = Padding (andar ka protection)
Cardboard box = Border (deewar)
Space between this box and other boxes = Margin (doori)

/* ===== BOX MODEL ===== */ .box { /* CONTENT */ width: 300px; height: 200px; /* PADDING — Andar ka space (content se border tak) */ padding: 20px; /* Sab taraf 20px */ padding: 10px 20px; /* Top-Bottom 10px, Left-Right 20px */ padding: 10px 20px 15px 25px; /* Top Right Bottom Left (clockwise) */ /* BORDER — Box ki deewar */ border: 3px solid #2563eb; /* width style color */ border-radius: 15px; /* Corner round karna */ border-top: 5px dashed red; /* Sirf top border alag */ /* MARGIN — Bahar ka space */ margin: 30px auto; /* auto = center horizontally */ margin-top: 20px; margin: 10px 15px 20px 25px; /* Top Right Bottom Left */ /* ===== BOX-SIZING — SABSE IMPORTANT ===== */ box-sizing: border-box; /* content-box (default): width = sirf content border-box: width = content + padding + border Hamesha border-box use karo! */ } /* ===== DISPLAY PROPERTIES ===== */ .block { display: block; } /* Poori width lega (div, p, h1) */ .inline { display: inline; } /* Sirf content jitni width (span, a) */ .inline-block { display: inline-block; } /* Inline + width/height set kar sakte hain */ .none { display: none; } /* Hide karna */
Warning: Hamesha box-sizing: border-box use karo — ye layout calculations easy kar deta hai! Bina iske width sirf content ki hoti hai, padding aur border alag add hote hain jo calculations mushkil banate hain.

Flexbox — Modern Layout King

Importance:
100% — 1D Layouts ke liye Best

Logic Samjho:

Flexbox = 1-dimensional layout system (row ya column). Parent ko display: flex karo, aur children automatically align honge. Ye responsive design ka base hai. Har modern website mein flexbox use hota hai.

Real Life Example:

Ek bookshelf socho jisme books automatically adjust hoti hain — chhoti jagah mein chhoti books, badi jagah mein badi books. Flexbox waise hi elements ko available space ke hisaab se adjust karta hai.

CSS Grid — 2D Layout Powerhouse

Importance:
95% — Complex Layouts ke liye

Logic Samjho:

CSS Grid = 2-dimensional layout system (rows + columns dono). Complex layouts jaise dashboards, photo galleries, aur page layouts ke liye best hai. Flexbox 1D hai (sirf row ya column), Grid 2D hai (rows aur columns dono ek saath).

Real Life Example:

Excel spreadsheet socho — rows aur columns ka combination. Grid bhi waise hi kaam karta hai. Flexbox = ek line mein books arrange karna, Grid = chess board jaise arrange karna.

/* ===== CSS GRID PARENT ===== */ .grid-container { display: grid; /* ===== COLUMNS DEFINE KARNA ===== */ grid-template-columns: 200px 200px 200px; /* 3 fixed columns */ grid-template-columns: 1fr 1fr 1fr; /* 3 equal flexible columns */ grid-template-columns: repeat(3, 1fr); /* Same as above — shorthand */ grid-template-columns: 200px 1fr 2fr; /* Fixed + Flexible mix */ /* ===== RESPONSIVE GRID — AUTO-FIT + MINMAX ===== */ /* Ye sabse behtar tarika hai responsive grid banane ka! */ grid-template-columns: repeat(auto-fit, minmax(250px, 1fr)); /* auto-fit: Jitni jagah hai utni columns minmax(250px, 1fr): Minimum 250px, maximum equal share Screen chhoti hogi toh columns kam ho jayengi */ /* ===== ROWS DEFINE KARNA ===== */ grid-template-rows: 100px auto 100px; /* Fixed + Auto + Fixed */ grid-template-rows: repeat(3, 1fr); /* 3 equal rows */ /* ===== GAPS ===== */ gap: 20px; /* Row aur column dono */ row-gap: 15px; /* Sirf rows mein */ column-gap: 25px; /* Sirf columns mein */ /* ===== NAMED AREAS (Advanced) ===== */ grid-template-areas: "header header header" "sidebar main main" "footer footer footer"; } /* ===== GRID CHILDREN ===== */ .header { grid-area: header; } .sidebar { grid-area: sidebar; } .main { grid-area: main; } .footer { grid-area: footer; } /* ===== SPANNING — Items ko merge karna ===== */ .wide-item { grid-column: 1 / 3; /* Column 1 se 3 tak (span 2) */ /* grid-column: 1 / span 2; */ /* Same thing */ grid-row: 1 / 2; /* Row 1 se 2 tak */ } /* ===== JUSTIFY & ALIGN ITEMS ===== */ .grid-container { justify-items: center; /* Horizontal center */ align-items: center; /* Vertical center */ justify-content: center; /* Grid ko container mein center */ align-content: center; /* Grid ko vertically center */ }
Tip: Responsive Grid ke liye auto-fit + minmax combination use karo — ye automatically columns adjust karta hai screen size ke hisaab se! Ye 2026 mein sabse popular technique hai.

Positioning & Z-Index

Importance:
90% — Overlays & Modals ke liye

Logic Samjho:

Positioning elements ko normal flow se bahar nikaal ke kahi aur place karti hai. Z-index stacking order control karta hai — kaunsa element upar rahega. Higher z-index = upar dikhega.

Real Life Example:

Ek restaurant mein socho — normal tables = static position. VIP section alag jagah pe = absolute. Waiter table ke paas khada = fixed. Menu card table pe = relative. Stacking = plates ek ke upar ek.

/* ===== POSITION TYPES ===== */ .static { position: static; /* Default — normal flow mein */ /* top, left, right, bottom — kaam nahi karenge */ } .relative { position: relative; /* Normal position se shift */ top: 20px; /* Neeche 20px (apni original position se) */ left: 30px; /* Right 30px */ /* Original space reserved rehta hai */ } .absolute { position: absolute; /* Nearest positioned parent se relative */ top: 0; /* Parent ke top se 0px */ right: 0; /* Parent ke right se 0px */ /* Parent ko position: relative; karna padta hai */ } .fixed { position: fixed; /* Viewport (screen) se fixed */ bottom: 20px; /* Screen ke bottom se 20px */ right: 20px; /* Screen ke right se 20px */ /* Scroll ke saath move nahi hoga */ } .sticky { position: sticky; /* Scroll ke saath stick hota hai */ top: 0; /* Top pe chipak jayega jab scroll karega */ /* Parent ke andar sticky rehta hai */ } /* ===== Z-INDEX — STACKING ORDER ===== */ .modal { position: fixed; top: 0; left: 0; width: 100%; height: 100%; background: rgba(0,0,0,0.5); z-index: 1000; /* Sabse upar dikhana hai */ } .tooltip { position: absolute; z-index: 100; /* Modal se neeche, baaki sab se upar */ } /* z-index sirf positioned elements pe kaam karta hai (relative, absolute, fixed, sticky) */
